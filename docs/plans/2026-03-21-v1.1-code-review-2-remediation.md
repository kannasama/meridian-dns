## Task 4: CR2-SEC-04 — Add SubprocessProvider response size limit

The stdout read loop in `SubprocessProvider.cpp:119-128` grows `sResponse` unboundedly if the subprocess emits a long line with no newline. Add a 16 MB guard.

**Files:**
- Modify: `src/providers/SubprocessProvider.cpp:119-128`

**Step 1: Add size limit to the read loop**

In `src/providers/SubprocessProvider.cpp`, find the read loop (lines ~119-128):

```cpp
  // ── Read one JSON line from child's stdout ────────────────────────────────
  std::string sResponse;
  char aBuf[4096];
  ssize_t nRead = 0;
  while ((nRead = read(aStdoutPipe[0], aBuf, sizeof(aBuf))) > 0 ||
         (nRead < 0 && errno == EINTR)) {
    if (nRead < 0) continue;  // EINTR, retry
    sResponse.append(aBuf, static_cast<size_t>(nRead));
    if (sResponse.find('\n') != std::string::npos) break;
  }
```

Replace with:

```cpp
  // ── Read one JSON line from child's stdout ────────────────────────────────
  static constexpr size_t kMaxResponseBytes = 16 * 1024 * 1024;  // 16 MB
  std::string sResponse;
  char aBuf[4096];
  ssize_t nRead = 0;
  while ((nRead = read(aStdoutPipe[0], aBuf, sizeof(aBuf))) > 0 ||
         (nRead < 0 && errno == EINTR)) {
    if (nRead < 0) continue;  // EINTR, retry
    sResponse.append(aBuf, static_cast<size_t>(nRead));
    if (sResponse.size() > kMaxResponseBytes) {
      close(aStdoutPipe[0]);
      waitpid(pid, nullptr, 0);
      throw common::ProviderError("SUBPROCESS_RESPONSE_TOO_LARGE",
                                  "Subprocess response exceeds 16 MB limit");
    }
    if (sResponse.find('\n') != std::string::npos) break;
  }
```

Note: `pid` and `aStdoutPipe` are local variables in the `callSubprocess()` method. Verify the variable names match the surrounding code before applying.

## Security: Outbound Request Considerations (SSRF)

Meridian DNS providers (PowerDNS, Cloudflare, DigitalOcean, Generic REST, Subprocess) make
outbound HTTP requests to admin-configured API endpoints. This is by design — DNS provider
management requires external API access.

**Cloud metadata services:** When deploying on cloud infrastructure (AWS, GCP, Azure), provider
endpoints could theoretically be configured to target the instance metadata service
(`169.254.169.254`). Since provider configurations are restricted to admin users, this is an
accepted risk scoped to the admin trust boundary.

To mitigate in sensitive environments:
- Use network-level controls (iptables, security groups) to block outbound traffic to
  `169.254.169.254` from the Meridian container
- Use IMDSv2 (AWS) which requires a PUT-based token exchange that HTTP GET-based SSRF cannot
  perform
- Restrict the `Admin` role to trusted users who would already have infrastructure access
```

---

## 7. Thread Safety

Functions that use static internal buffers are **banned** in this project. All code runs in a
multi-threaded context (Crow worker threads, ThreadPool, MaintenanceScheduler).

### Banned Functions and Replacements

| Banned | Replacement | Header |
|--------|------------|--------|
| `std::gmtime()` | `gmtime_r()` via `common/TimeUtils.hpp` | `#include "common/TimeUtils.hpp"` |
| `std::localtime()` | `localtime_r()` | POSIX `<time.h>` |
| `std::asctime()` | `strftime()` with `gmtime_r()` result | POSIX `<time.h>` |
| `std::ctime()` | `strftime()` with `localtime_r()` result | POSIX `<time.h>` |
| `std::strtok()` | Manual `find()`/`substr()` or `std::string_view` | N/A |
| `getenv()` in hot paths | Cache at startup | N/A |

**Standard pattern for timestamps:**

```cpp
#include "common/TimeUtils.hpp"

// Preferred — use the project utility
std::string sTs = dns::common::TimeUtils::nowIso8601();

// If custom format is needed — use gmtime_r directly
std::tm tmBuf{};
gmtime_r(&tTime, &tmBuf);
std::ostringstream oss;
oss << std::put_time(&tmBuf, "%Y-%m-%dT%H:%M:%SZ");
```

> **History:** `std::gmtime()` was caught twice in code review (CR-CRYPTO-01 in v1.0,
> CR2-SEC-01 in v1.1). This standard exists to prevent recurrence.

---

## 8. SQL and Database

### Parameterized Queries

All SQL queries MUST use libpqxx parameterized queries (`pqxx::params{...}`) with `$N`
placeholders. Never interpolate user input into SQL strings.

### Array Construction

When building PostgreSQL array literals for use in SQL, use `txn.quote()` for each element.
**Do not manually escape** quotes or backslashes.

**Correct:**

```cpp
std::string sArr = "ARRAY[";
for (size_t i = 0; i < vItems.size(); ++i) {
  if (i > 0) sArr += ",";
  sArr += txn.quote(vItems[i]);
}
sArr += "]";
```

**Incorrect (banned):**

```cpp
// Do NOT do this — fragile manual escaping
for (char c : sItem) {
  if (c == '"' || c == '\\') sEscaped += '\\';
  sEscaped += c;
}
```

> **History:** Manual array escaping in `TagRepository` was flagged in CR2-SEC-05 (v1.1).
> Use `txn.quote()` consistently, as `ZoneRepository::updateTags()` does.
```
